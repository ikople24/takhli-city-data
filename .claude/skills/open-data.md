# ทักษะ: Open Data สำหรับเทศบาล

## ชุดข้อมูลที่เทศบาลควรเปิดเผย (Priority List)

### Tier 1 — เปิดทันที (พร้อมอยู่แล้ว ความเสี่ยงต่ำ)
```
□ งบประมาณรายจ่ายประจำปี (แยกหมวด/กอง)
□ ผลการจัดซื้อจัดจ้าง (รายโครงการ)
□ แผนพัฒนาท้องถิ่น พ.ศ. 2566–2570
□ รายงานผลการดำเนินงานรายปี
□ ข้อมูลบริการประชาชน (ประเภท ขั้นตอน เวลา)
□ ตำแหน่งสถานที่สำคัญ (GeoJSON)
□ แหล่งท่องเที่ยวและ OTOP
```

### Tier 2 — เปิดหลังเตรียมข้อมูล (ต้องทำความสะอาดก่อน)
```
□ สถิติการร้องเรียน (รายเดือน แยกประเภท ไม่มีชื่อ)
□ ปริมาณขยะที่จัดเก็บ (รายวัน/เดือน แยกชุมชน)
□ คุณภาพน้ำประปา (ผลตรวจรายเดือน)
□ สถิติผู้ใช้บริการแต่ละกอง
□ ข้อมูลผู้สูงอายุ/คนพิการ (เฉพาะสถิติรวม ไม่มีชื่อ)
□ ปริมาณการใช้น้ำประปา (รายเดือน รวมทั้งเขต)
```

### Tier 3 — เปิดเมื่อระบบ Smart City พร้อม
```
□ ข้อมูล Sensor สิ่งแวดล้อม (real-time)
□ ข้อมูลจราจร/การเดินทาง (รวม Aggregate)
□ การใช้พลังงานอาคารสาธารณะ
□ ดัชนีคุณภาพอากาศ (AQI)
```

## รูปแบบไฟล์ข้อมูล

| รูปแบบ | ใช้กับ | ข้อดี |
|--------|--------|-------|
| CSV | ตาราง/สถิติ | เปิดได้ทุก software |
| JSON/GeoJSON | ข้อมูลเชิงพื้นที่ / API | Machine-readable |
| XLSX | รายงานที่ต้องคำนวณ | คนทั่วไปใช้ง่าย |
| PDF | เอกสารทางการ | ไม่เปลี่ยนรูปแบบ |

> กฎ: ทุก dataset ต้องมี CSV หรือ JSON เสมอ — PDF อย่างเดียวไม่นับว่าเป็น Open Data

## มาตรฐาน Open Data 5 Stars (Tim Berners-Lee)

```
★     เผยแพร่บนเว็บ (ไม่ว่ารูปแบบใด)
★★    ข้อมูลมีโครงสร้าง (Excel แทน PDF scan)
★★★   รูปแบบที่ไม่ผูกติดกับ vendor (CSV แทน .xls)
★★★★  ใช้ URI ระบุแต่ละ record (Linked Data)
★★★★★ เชื่อมโยงกับ dataset อื่น (Linked Open Data)

เป้าหมายระยะแรก: ★★★ (เพียงพอสำหรับ ITA และประชาชน)
```

## Open Data Roadmap เทศบาลเมืองตาคลี

```
ปีที่ 1: เปิด Tier 1 ทั้งหมดบน data.go.th (6 ชุดข้อมูล)
         → รูปแบบ CSV + metadata ครบถ้วน

ปีที่ 2: เปิด Tier 2 + สร้าง Data Portal บนเว็บเทศบาล
         → อัปเดตอัตโนมัติ รายเดือน

ปีที่ 3: เปิด Tier 3 + REST API สำหรับนักพัฒนา
         → real-time sensor data
```

## การเผยแพร่บน data.go.th

### ขั้นตอน
1. สมัคร account หน่วยงานที่ data.go.th
2. เตรียมข้อมูลในรูปแบบ CSV / JSON ที่สะอาด
3. กรอก metadata ตามมาตรฐาน DCAT-AP TH
4. อัปโหลดและกำหนดสิทธิ์ (CC BY 4.0 แนะนำ)
5. เผยแพร่และอัปเดตตามรอบที่กำหนด

### เกณฑ์ ITA ที่เกี่ยวข้อง
- ตัวชี้วัดที่ O10: แผนดำเนินงานและการใช้งบประมาณ
- ตัวชี้วัดที่ O19: รายงานผลการจัดซื้อจัดจ้าง
- ตัวชี้วัดที่ O25: นโยบายการบริหารทรัพยากรบุคคล

## Data Quality Checklist ก่อนเผยแพร่
- [ ] ไม่มีข้อมูลส่วนบุคคล (ชื่อ เลขบัตร เบอร์โทร)
- [ ] Header ภาษาอังกฤษหรือภาษาไทยชัดเจน (ไม่มี col1, col2)
- [ ] ไม่มี cell ว่างโดยไม่มีเหตุผล (ใช้ null หรือ N/A)
- [ ] รูปแบบวันที่สม่ำเสมอ (YYYY-MM-DD)
- [ ] ตัวเลขไม่มี comma คั่นหลักพัน (ใช้ plain number)
- [ ] Encoding UTF-8 (รองรับภาษาไทย)
- [ ] มี README/คำอธิบายตัวแปรทุกคอลัมน์
