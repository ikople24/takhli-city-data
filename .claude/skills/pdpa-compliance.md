# ทักษะ: PDPA Compliance สำหรับเทศบาล

## ข้อมูลส่วนบุคคลที่เทศบาลเก็บ (แยกตามระบบ)

| ระบบ/กิจกรรม | ข้อมูลที่เก็บ | ฐานทางกฎหมาย |
|-------------|------------|-------------|
| ทะเบียนราษฎร/ประชากร | ชื่อ ที่อยู่ เลขบัตร วันเกิด | หน้าที่ตามกฎหมาย |
| กล้อง CCTV | ภาพใบหน้า ทะเบียนรถ | ประโยชน์สาธารณะ / ประโยชน์โดยชอบด้วยกฎหมาย |
| แอป/LINE OA | ชื่อ เบอร์ LINE ID | ความยินยอม |
| แบบฟอร์มร้องเรียน | ชื่อ ที่อยู่ เบอร์โทร | หน้าที่ตามกฎหมาย |
| เบี้ยยังชีพ/สวัสดิการ | ชื่อ เลขบัตร บัญชีธนาคาร สุขภาพ | หน้าที่ตามกฎหมาย |
| ภาษีที่ดิน | ชื่อ ที่อยู่ ข้อมูลทรัพย์สิน | หน้าที่ตามกฎหมาย |
| IoT / Sensor ชุมชน | ข้อมูลพฤติกรรม การเดินทาง | ประโยชน์สาธารณะ + แจ้งประชาชน |

## ฐานทางกฎหมายที่เทศบาลใช้บ่อย (มาตรา 24)

```
1. ความยินยอม (Consent) — ต้องชัดแจ้ง ถอนได้ ไม่บังคับ
   → ใช้กับ: แอปพลิเคชัน, Line OA, รับข่าวสาร

2. หน้าที่ตามกฎหมาย (Legal Obligation)
   → ใช้กับ: ทะเบียนราษฎร, ภาษี, สวัสดิการ, คำร้อง

3. ประโยชน์สาธารณะ (Public Task) — อ้างอิงอำนาจตาม พ.ร.บ.เทศบาล
   → ใช้กับ: CCTV ความปลอดภัย, IoT สิ่งแวดล้อม, GIS

4. ประโยชน์โดยชอบด้วยกฎหมาย (Legitimate Interest)
   → ใช้กับ: วิเคราะห์ข้อมูลปรับปรุงบริการ (ต้องทำ LIA ก่อน)
```

## DPIA (Data Protection Impact Assessment) — เมื่อไรต้องทำ

**ต้องทำ DPIA ก่อนเริ่ม project เมื่อ:**
- ติดตั้ง CCTV ครอบคลุมพื้นที่สาธารณะ ✅
- ระบบจดจำใบหน้า (Face Recognition) ✅
- IoT ติดตามพฤติกรรม/การเดินทางประชาชน ✅
- Big Data วิเคราะห์โปรไฟล์ประชาชน ✅
- ระบบที่เกี่ยวข้องกับเด็ก ✅

**โครงสร้าง DPIA อย่างย่อ:**
```
1. คำอธิบาย project และข้อมูลที่ใช้
2. ความจำเป็นและสัดส่วน (จำเป็นแค่ไหน เก็บน้อยที่สุด?)
3. ความเสี่ยงต่อสิทธิและเสรีภาพ (สูง/กลาง/ต่ำ)
4. มาตรการลดความเสี่ยง
5. บทสรุปและการอนุมัติ
```

## บันทึกกิจกรรมการประมวลผล (ROPA)

```
| ชื่อกิจกรรม | วัตถุประสงค์ | ประเภทข้อมูล | ฐานกฎหมาย | ระยะเวลาเก็บ | ผู้เข้าถึง |
```

## โทษตาม พ.ร.บ. PDPA

| ความผิด | โทษสูงสุด |
|--------|---------|
| เก็บข้อมูลโดยไม่มีฐานกฎหมาย | ปรับ 3 ล้านบาท |
| เปิดเผยข้อมูลโดยไม่ได้รับอนุญาต | จำคุก 6 เดือน + ปรับ 500,000 บาท |
| ข้อมูลอ่อนไหว (สุขภาพ ศาสนา) ละเมิด | ปรับ 5 ล้านบาท |
| ไม่แจ้ง Data Breach ภายใน 72 ชั่วโมง | ปรับ 3 ล้านบาท |

## Checklist ก่อน Launch Smart City Project
- [ ] ระบุข้อมูลส่วนบุคคลที่เก็บทั้งหมด
- [ ] เลือกฐานทางกฎหมายที่เหมาะสม
- [ ] ทำ DPIA (ถ้า project มีความเสี่ยงสูง)
- [ ] จัดทำ Privacy Notice และเผยแพร่
- [ ] บันทึก ROPA
- [ ] กำหนด DPO และช่องทางรับคำร้อง
- [ ] ฝึกอบรมเจ้าหน้าที่ที่เกี่ยวข้อง
- [ ] วางแผน Data Breach Response
